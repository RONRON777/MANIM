
AI 코딩 지침서 (생산 버전)
================================

이 문서는 AI가 코드 생성을 수행할 때 따라야 할 **실제 사용 가능한** 규칙을 정의합니다. 아래 규칙을 준수하여 읽기 쉽고 유지보수가 용이하며 안전한 코드를 생성하십시오.

1. **프롬프트 구조**
   - 사용자가 요청하는 작업을 한 줄 요약으로 명확히 설명하고, 입력 데이터나 추가 지시 사항은 `"""` 또는 `###` 등 구분자로 나누어 제공한다.
   - 프롬프트에는 프로그래밍 언어와 필요한 라이브러리를 명시한다. 예를 들어 Python 코드를 생성할 때는 `import` 문을 먼저 작성하여 언어를 지정한다.
   - 모호한 표현을 피하고, 결과 형식(예: 반환 값의 유형, 출력의 길이, 포맷)을 구체적으로 명시한다.
   - AI가 익숙하지 않은 API나 라이브러리를 사용할 경우, API 정의와 간단한 사용 예제를 프롬프트에 포함하도록 요구한다.

2. **코딩 스타일**
   - **네이밍 규칙**: 변수와 함수는 `snake_case`(소문자와 언더스코어)로, 클래스는 `PascalCase`로 작성한다. 이름은 기능을 설명하는 의미 있는 단어로 구성한다.
   - **들여쓰기 및 줄 길이**: 4칸 공백 들여쓰기를 사용하고, 한 줄은 100자를 넘기지 않는다. 탭 대신 공백을 사용한다.
   - **주석과 문서**: 함수, 클래스, 모듈에는 docstring을 작성하여 목적, 매개변수, 반환값을 설명한다. 주석은 코드의 의도가 불분명한 부분에만 추가하고, 코드가 **왜** 그런 방식으로 작성되었는지 설명한다.
   - **모듈화**: 하나의 함수나 클래스는 하나의 책임만을 가지도록 하고, 복잡한 로직을 작은 단위로 분리한다. 깊은 중첩은 피하고 재사용 가능한 모듈로 구성한다.

3. **에러 처리**
   - 예상 가능한 모든 예외를 포착하여 적절한 메시지를 제공한다. 가능한 한 구체적인 예외 유형을 사용하고, 예외 발생 시 프로그램이 안전하게 종료되거나 복구할 수 있도록 한다.
   - 사용자 입력은 검증하여 SQL 인젝션, XSS 등 보안 취약점을 방지한다. 민감한 정보를 직접 코드에 포함하거나 출력하지 않는다.

4. **보안 및 민감 정보 보호**
   - API 키, 비밀번호, 개인식별정보 등 민감한 데이터를 코드나 프롬프트에 노출하지 않는다.
   - 외부 네트워크 호출이나 파일 시스템 변경과 같이 부작용을 일으킬 수 있는 동작은 사용자로부터 명시적인 요청이 없는 한 포함하지 않는다.
   - 암호화나 인증 로직을 구현할 때는 검증된 라이브러리를 사용하고, 자체 구현을 지양한다.

5. **테스트와 검토**
   - 핵심 기능에는 단위 테스트를 작성한다. Python은 `pytest`, JavaScript는 `jest` 등 언어에 적합한 테스트 프레임워크를 사용한다.
   - 작성된 코드는 다른 개발자가 검토할 수 있도록 구조화하고, 테스트 코드가 실제 동작을 검증하도록 한다. AI가 생성한 코드라도 사람이 반드시 리뷰해야 한다.

6. **성능과 최적화**
   - 코드 작성 시 우선순위는 가독성과 유지보수성이다. 성능 최적화는 필요하다고 판단될 때 프로파일러를 사용하여 병목을 확인한 후 수행한다.
   - 반복적으로 호출되는 연산에는 캐싱을 고려하되, 캐시 무효화 전략을 함께 설계한다.

7. **규칙 준수**
   - 사용자 요구 사항을 벗어난 추가 기능이나 출력을 생성하지 않는다. 명시적인 질문이나 요구가 있을 때만 세부 구현을 확장한다.
   - 사용자의 지시와 충돌하는 규칙이 있을 경우, 안전과 보안을 최우선으로 하여 적절히 조정하거나 사용자에게 확인을 요청한다.
   - 코드 출력 시 불필요한 설명을 포함하지 않고, 요청된 내용에 맞추어 순수한 코드만을 제공한다(예외: 주석이나 docstring이 요청에 포함된 경우).

이 지침서는 AI가 고품질 코드를 생성하도록 돕기 위해 설계되었습니다. AI는 위 규칙을 따르며, 사용자가 제공한 명세에 기반해 정확하고 신뢰할 수 있는 코드를 작성해야 합니다.
